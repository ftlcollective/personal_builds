Security Concepts

What this project teaches.

Every decision in this project is documented — not just what the code does, but why. Security is about thinking, not just syntax.

Attack Vector

Cross-Site Scripting (XSS)

XSS attacks inject malicious scripts into web pages through user input fields. When another user views that page, the script executes — potentially stealing session data, cookies, or credentials. SecureForm demonstrates how to prevent this at the input level.

Defence

Input Sanitisation

Every character typed by a user is treated as untrusted. Before it touches any function or storage, it runs through a sanitiser that strips dangerous patterns — script tags, event handlers, and JavaScript protocol strings.

Validation Layer

Structure Validation

After sanitisation, fields are validated for correct format — email patterns, length limits, and character allowlists. This second layer catches malformed input that survived sanitisation.

Output Layer

Output Encoding

Any user-supplied data that is displayed back to the browser is HTML-encoded first. Even if something slips through sanitisation, encoding prevents it from being interpreted as markup or script by the browser.

Defence in Depth

Three layers of protection.

01

Sanitise on input

Every keystroke triggers the sanitiser. Never wait until submit — attack vectors are stripped the moment they're typed, before they can reach any function or storage.

02

Validate structure

After sanitisation, fields are tested for correct format. Email patterns, length limits, and allowed character sets — a clean input that doesn't match structure is still rejected.

03

Encode on output

Any data shown back to the user is HTML-encoded before rendering. Even if something slips past layers one and two, the browser will never interpret it as executable code.
