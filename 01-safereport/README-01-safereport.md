## 📌 About This Project

SafeReport is a **privacy-first web application** for anonymous reporting and resource access, built for individuals experiencing gender-based violence.

This project was built at the intersection of my work as a **qualified Women's Health Practitioner** and my development as a **full-stack developer**. It is not a generic form — it is a considered, empathy-driven tool designed around the real needs of survivors.

The core design principle: **every decision protects the user.** No data is collected. No personal information is required. No barriers to access.

---

## 🔐 Privacy Architecture

| Feature | Implementation |
|---|---|
| No IP logging | Session is fully stateless — no server-side tracking |
| No cookies | Zero persistent storage set by the application |
| No personal data required | Every field on the report form is optional |
| End-to-end approach | Input is sanitised and handled client-side only |
| Safe exit | Tab can be closed at any point — no data persists |

> 🔒 **Anonymous session active.** We do not log IP addresses, device identifiers, or any personal information.

---

## ✨ Features

- **Anonymous report submission** — multi-field form with zero required fields
- **Integrated helpline directory** — verified South African GBV support organisations
- **Mobile-first responsive design** — accessible on any device, including low-end phones
- **Sanitised input handling** — XSS prevention applied to all form inputs
- **No backend dependency** — fully client-side, deployable anywhere

## 💡 What I Learned

- Designing for **emotional safety** as well as technical security — tone, language, and layout all matter when your user may be in crisis
- Implementing **input sanitisation** without a backend framework — pure JS XSS prevention
- Building forms that feel **calm and non-threatening** — a UX challenge that goes beyond aesthetics
- How to structure a **stateless application** that leaves no trace
